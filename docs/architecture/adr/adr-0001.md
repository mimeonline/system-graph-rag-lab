# ADR-0001 Deployment Target Vercel und Neo4j Aura

## Status
1. Accepted am 2026-02-25.

## Kontext
1. Das MVP muss öffentlich erreichbar sein und schnell iterierbar bleiben.
2. Das Team benötigt eine einfache Betriebsbasis ohne eigene Infrastruktur.
3. Der Graph Speicher muss für kleine Demo Datensätze verwaltet verfügbar sein.
4. OpenAI API Zugriff erfolgt serverseitig mit Budget Limit.

## Entscheidung
1. Web UI und API Layer laufen als eine gemeinsame Next.js Anwendung auf Vercel.
2. Wissensgraph und Vektorindex laufen auf Neo4j Aura.
3. Quellcode liegt im öffentlichen GitHub Repository.
4. Secrets werden ausschließlich über Runtime Environment Variables gesetzt.
5. Lokale Secret und Key Werte werden über `.env.local` und optional `.env` bereitgestellt und nicht versioniert.
6. Public Secret und Key Werte werden ausschließlich als Vercel Environment Variables bereitgestellt.

## Begründung
1. Vercel reduziert Betriebsaufwand für Public Demo Deployment.
2. Neo4j Aura bietet verwalteten Betrieb für Graph Workload im MVP Umfang.
3. Die Kombination passt zum bestehenden Betriebsrahmen in den Discovery Artefakten.
4. Die Entscheidung minimiert Setup Zeit für Dev und QA.

## Konsequenzen
1. API Runtime muss als Next.js Route Handler stateless und serverless tauglich implementiert werden.
2. Rate Limiting braucht eine serverless kompatible Strategie.
3. Logs und Metriken bleiben zunächst auf minimalem Runtime Niveau.
4. Lokale und Cloud Konfigurationen müssen dieselben Contracts einhalten.
5. Repository Historie darf keine echten Secret oder Key Werte enthalten.

## Verworfene Alternativen
1. Eigenes Container Hosting erhöht Betriebsaufwand und ist für MVP nicht nötig.
2. Selbst verwaltete Neo4j Instanz erhöht Komplexität und Betriebsrisiko.
