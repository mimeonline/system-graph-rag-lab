# ADR-0002 Retrieval Strategy Contract

## Status
1. Accepted am 2026-02-25.

## Kontext
1. Das MVP benötigt reproduzierbares Retrieval für Dev Umsetzung und QA Tests.
2. Der Graph hat mehr als 100 Nodes und mehr als 200 Edges, daher muss die Strategie deterministisch und effizient bleiben.
3. Token Kosten müssen für Public Demo begrenzt werden.

## Entscheidung
1. `TopK` für Seed Retrieval ist fix auf `6`.
2. `Hop Depth` für Graph Expansion ist fix auf `1`.
3. `Context Budget` für Retrieval Kontext ist fix auf `1400` Tokens.
4. Maximal `8` Evidenzknoten werden an den LLM Kontext übergeben.
5. Dedupe erfolgt strikt über `nodeId`.
6. Stabile Sortierung erfolgt nach `score DESC`, `hop ASC`, `nodeType ASC`, `nodeId ASC`.

## Begründung
1. TopK 6 deckt im kleinen Graph ausreichend semantische Seeds ab.
2. Hop Depth 1 begrenzt Rauschen und hält Antwortqualität im MVP kontrollierbar.
3. Context Budget 1400 begrenzt Kosten und reduziert Overflow Risiko.
4. Ein fester Sortierschlüssel macht QA Replays deterministisch.

## Konsequenzen
1. Tieferes Multihop Reasoning bleibt außerhalb des MVP.
2. Relevanz wird stark vom Seed Ranking und von Knotensummaries beeinflusst.
3. Änderungen an TopK, Hop Depth oder Budget erfordern neue ADR Entscheidung.

## Verworfene Alternativen
1. Dynamisches TopK erhöht Variabilität und erschwert QA.
2. Hop Depth 2 als Default erhöht Tokenverbrauch ohne garantierten Mehrwert im kleinen Graph.
3. Unbudgetierter Kontext erzeugt Kostenrisiko für Public Traffic.
