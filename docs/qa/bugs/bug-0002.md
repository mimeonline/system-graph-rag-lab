# Bug 0002 Story-Testkommando fuehrt falschen Scope aus

## Status
1. Resolved am 2026-02-26.
2. Urspruenglich Open am 2026-02-25 im QA-Run `RUN_MODE=review` fuer `E1-S4`.

## Titel
1. Das im Dev-Handoff dokumentierte Kommando `pnpm --dir apps/web test -- src/features/seed-data/quality-check.test.ts` fuehrt im aktuellen Setup die Gesamtsuite statt nur des Story-Testfiles aus.

## Schweregrad
1. Low.

## Repro Steps
1. Im Repository ausfuehren: `pnpm --dir apps/web test -- src/features/seed-data/quality-check.test.ts`.
2. Testausgabe pruefen.
3. Vergleich ausfuehren: `pnpm --dir apps/web exec vitest run src/features/seed-data/quality-check.test.ts`.

## Expected vs Actual
1. Expected: Das Story-Kommando testet nur `src/features/seed-data/quality-check.test.ts`.
2. Actual: Das Kommando startet die komplette Suite mit mehreren Testdateien.
3. Actual Vergleich: Der direkte Vitest-Run prueft nur die eine Story-Datei mit 3 Tests.

## Root Cause
1. Das Kommando wurde ueber den Package-Script-Aufruf mit Separator `--` dokumentiert.
2. In dieser Konfiguration wurde der Dateiscope nicht deterministisch nur auf das Ziel-Testfile begrenzt.
3. Dadurch war die Story-spezifische Evidenz unnoetig breit und weniger reproduzierbar.

## Fix
1. Story-spezifisches QA-Kommando wurde auf direkten Vitest-Aufruf normiert:
   `pnpm --dir apps/web exec vitest run src/features/seed-data/quality-check.test.ts`.
2. Dev-Handoff fuer `E1-S4` verwendet jetzt dieses exakte Scope-Kommando.
3. QA-to-DevOps-Handoff verwendet ebenfalls das direkte Scope-Kommando.

## Logs oder Screenshots Referenzen
1. QA-Command-Evidenz im Story-Gate `docs/qa/verdict.md`.
2. Dev-Handoff-Referenz `docs/handoff/dev-to-qa.md`.

## Workaround
1. Kein weiterer Workaround noetig; der Fix ist in den Handoff-Artefakten uebernommen.
